# IQ Buddy Agent Demo 1

IQ Buddy / Workflow Studio 的静态交互原型，用于演示硅基实习生工作台、侧导航会话历史、权限申请详情，以及分栏、抽屉、弹窗等不同信息承载形态。

## 在线访问

- GitHub 仓库: https://github.com/monica137142/iqbuddy-agent-demo1
- GitHub Pages: 如仓库已开启 Pages，可访问 `https://monica137142.github.io/iqbuddy-agent-demo1/`

## 项目内容

- `index.html`: 入口页，自动跳转到主原型页面。
- `agents-replica.html`: 主演示页面，包含侧导航、硅基实习生列表、权限详情、抽屉/弹窗/分栏形态、会话历史和自动化任务等交互。
- `public-agents.json`: Agent 数据配置。
- `source-index.css`: 源页面样式参考。
- `source-index.js`: 源页面脚本参考。
- `vendor-antd.js`: 本地依赖脚本。
- `DESIGN.md`: 设计系统说明。

## 主要功能

- 侧导航包含三个硅基实习生入口：
  - `硅基实习生-抽屉`
  - `硅基实习生-分栏`
  - `硅基实习生-弹窗`
- 三个硅基实习生入口的筛选、搜索、详情打开状态和滚动位置相互独立。
- 实习生分类筛选拆分为连续业务分类组（`全部`、`种草业务线`、`内容运营`、`数据分析`、`会员运营`）和独立 `待上线` 状态按钮，按钮组为灰色底，选中态为白色按钮底，文字保持垂直居中；切换 `全体实习生` / `我的实习生` 时，二级分类默认回到 `全部`。
- `待上线 n` 筛选 tab 展示测试中的硅基实习生；前 5 个为黄色实底 `去训练` card 并出现在左侧会话/任务中，剩余均为默认透明、hover 浅黄的 `申请权限` card；点击 card 打开详情抽屉，点击按钮直接提交权限申请。
- 从实习生 card 新建对话或任务时，会自动定位左侧会话/任务列表中的对应实习生，使其出现在可视区域中部。
- 左侧 `会话/任务` 标题右侧提供 1.2px 列表式展开/收起图标，可统一展开或收起下方所有 agent 会话历史。
- `硅基实习生-分栏` 支持左右分栏展示权限详情，并可拖动中间分割线调整宽度，右侧比例不超过 1:1。
- `硅基实习生-抽屉` 支持点击申请权限卡片后以右侧抽屉蒙层展示权限详情。
- `硅基实习生-弹窗` 支持以弹窗展示权限详情，底部申请权限按钮固定在弹窗底部。
- 权限详情包含固定头部、Agent 头像名称、副标题、状态标签、简介、核心能力、适用场景、支持平台和地区表格、使用手册链接。
- 左侧会话/任务列表支持历史会话、更多折叠、展开收起、hover 新建、历史删除确认等交互。
- 无历史的会话类 Agent 可通过 `新建` 创建一个新的对话式任务表单，并进入历史列表。

## 本地预览

这是一个静态页面项目，可以直接打开 `index.html`，也可以用任意静态服务器预览：

```bash
python3 -m http.server 3000
```

然后访问：

```text
http://localhost:3000/
```

## 部署方式

项目为纯静态页面，推送到 GitHub 后可通过 GitHub Pages 或任意静态站点托管服务部署。入口文件为 `index.html`。

## 备注

该仓库主要用于原型评审和交互验证，不包含后端服务。页面中的数据和交互多为本地静态模拟。
